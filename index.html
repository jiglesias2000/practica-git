Conceptos Básicos de Git

¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite gestionar y registrar los cambios en archivos y proyectos de software.

Conceptos Clave

  • Repositorio: Es el lugar donde se almacena el historial de cambios de un proyecto. Puede ser local o remoto.
  • Commit: Es un registro de los cambios realizados en el repositorio. Cada commit tiene un mensaje descriptivo.
  • Branch (Rama): Es una línea de desarrollo independiente. Permite trabajar en nuevas funcionalidades sin afectar la rama principal.
  • Merge: Es la acción de combinar los cambios de diferentes ramas en una sola.
  • Clone: Copia un repositorio remoto a tu máquina local.
  • Push: Envía los commits locales al repositorio remoto.
  • Pull: Descarga y fusiona los cambios del repositorio remoto al local.
  • Staging Area: Zona intermedia donde se preparan los archivos antes de hacer un commit.

Comandos Básicos

  • git init: Inicializa un nuevo repositorio.
  • git clone <url>: Clona un repositorio remoto.
  • git status: Muestra el estado de los archivos.
  • git add <archivo>: Añade archivos al staging area.
  • git commit -m "mensaje": Guarda los cambios en el repositorio.
  • git push: Envía los cambios al repositorio remoto.
  • git pull: Descarga y fusiona cambios del remoto.
  • git branch: Lista las ramas existentes.
  • git checkout <rama>: Cambia de rama.
  • git merge <rama>: Fusiona una rama con la actual.
